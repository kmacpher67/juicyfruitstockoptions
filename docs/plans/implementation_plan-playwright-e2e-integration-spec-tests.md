# Infrastructure & UX Roadmap Refinement

## Context
Currently, the codebase does not have a dedicated section or file detailing CI/CD practices or GitHub Actions integration for Playwright. Testing automation is mostly focused on local verification workflows. Additionally, after navigating the current local build of the Juicy Fruit web application, I have gathered visual feedback to propose a modern, refined set of UI/UX feature requirements to bring the app to a "WOW" state.

## Proposed Changes

We will introduce a CI/CD framework section to the master roadmap and append several refined UI/UX items based on the visual inspection of the app.

---

### Phase 1: CI/CD & Automation Integration (F-R Addition)
*We should formally add this to `docs/features-requirements.md` under the Infrastructure section to ensure automated validation doesn't get left behind.*

#### [NEW] **CI/CD Pipeline Integration**
- [ ] **GitHub Actions / Docker CI Base**: Create a `.github/workflows/playwright.yml` file to handle automated pull request / push validations.
- [ ] **Headless Automation**: Configure the pipeline to run the Dockerized backend services (MongoDB, FastAPI) alongside the frontend, wait for ports to open, and then execute `npx playwright test`.
- [ ] **CI Test Artifacts**: Capture and store failed test traces, screenshots, and videos generated by Playwright so that any failed test can be interactively debugged via GitHub Actions artifacts.
- [ ] **Agent Workflow Guidance**: Introduce an `.agent/workflows/ci-cd.md` workflow file to guide agents on the rules around modifying CI/CD pipelines, handling mocked API data within tests, and maintaining the GitHub Action yamls.

---

### Phase 2: Refined UI/UX Enhancements (F-R Addition)
*The app currently has an excellent high-density data feel using ag-Grid and a cohesive dark mode. However, the data presentation lacks modern visual components such as graphing, and the table densities require optimization for readability. We should add these to the `docs/features-requirements.md` under the Dashboard & UX Features block.*

#### [NEW] **Visual Data Visualization**
- [ ] **Interactive Candlestick Charts**: Integrate `lightweight-charts` (TradingView) or `D3.js` strictly into the Ticker Modal's "Price Action" tab to upgrade it from empty/text-based into a functional trading chart.
- [ ] **Mini Sparklines**: Add price trend sparklines directly into the Analysis and Portfolio table rows for rapid visual trend assessments (turning raw numbers into immediate insight).

#### [NEW] **Perceived Performance & Feedback**
- [ ] **Skeleton Loading States**: Implement skeleton row/grid implementations for ag-Grid and Modal tabs to provide immediate structural feedback while data fetches, avoiding abrupt "loading spinner" walls.
- [ ] **Sync Progress Animation**: Replace the static "TWS Live" indicator with a subtle, non-intrusive "Syncing..." micro-animation / progress bar when background fetches execute.

#### [NEW] **Actionable Interaction & Spatial Optimization**
- [ ] **"One-Click Roll" Acceleration**: Make the "Smart Rolls" suggestions explicitly actionable; clicking a roll suggestion should pre-populate a sliding order-ticket drawer.
- [ ] **Contextual Detail Drawer**: Add a UI toggle to shift Ticker Details from a centered Modal into a **Right-Side Slide-out Drawer**, allowing users to maintain context over the main data grid while scanning details.
- [ ] **Sticky Table Columns**: Ensure the Ticker symbol and essential action buttons remain pinned as sticky columns when scrolling horizontally across very wide data tables.
- [ ] **Ag-Grid Column Chooser**: Provide a user-facing column chooser dropdown to allow operators to hide/show specific columns dynamically, allowing them to manage excessive horizontal scrolling.
