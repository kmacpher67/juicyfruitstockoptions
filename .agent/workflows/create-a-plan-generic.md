# Create A Plan

Before non-trivial implementation work, create an implementation plan in `docs/plans/`.

## Checklist

- [ ] Is the requirement captured in `docs/features-requirements.md`?
- [ ] Does the feature need a dedicated doc in `docs/features/`?
- [ ] Does the work depend on outside research that should be saved in `docs/learning/`?
- [ ] Are config, env vars, credentials, schedules, or secrets changing?
- [ ] Are data models, storage, imports, exports, or indexes changing?
- [ ] Are third-party contracts or source-of-truth rules changing?
- [ ] Are there security, privacy, or access-control implications?
- [ ] What are the main failure modes and exception paths?
- [ ] What tests will prove the change works?
- [ ] What documentation must be updated before the work is considered done?

## Plan Requirements

The implementation plan should include:
- goal
- scope
- related requirements
- assumptions
- risks
- config and environment changes
- data/storage impact
- implementation steps
- test plan
- documentation plan
- definition of done
